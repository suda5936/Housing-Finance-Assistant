# Phase 7 정책 RAG·근거 관리

## 1. 완료 범위

Phase 7에서는 정책 원문을 근거로만 답변할 수 있도록 다음 흐름을 구현했다.

```text
공식 출처 스냅샷
→ 도메인·해시·기간 검증
→ 의미 단위 청킹
→ 키워드 + 로컬 경량 벡터 검색
→ 지역·정책·기준일 필터
→ 인용 구간 생성
→ 규칙 엔진 결과와 근거 연결
→ 근거 부족·검토 미완료 시 공식 확인 필요
```

별도 벡터 데이터베이스, 유료 임베딩 API, LLM 호출을 추가하지 않았다. 현재 정책이 3개뿐인 MVP이므로 Python 메모리 안에서 재현 가능한 검색 인덱스를 생성한다. 정책 수와 원문 규모가 커지면 PostgreSQL 또는 전용 벡터 인덱스로 교체할 수 있도록 문서·청크·검색 결과 모델을 분리했다.

## 2. 공식 출처와 신뢰 상태

허용 도메인은 다음 두 개로 고정했다.

- `housing.seoul.go.kr`
- `nhuf.molit.go.kr`

다른 도메인을 가진 출처는 카탈로그 로딩 단계에서 거부한다. 각 스냅샷은 제목, 기관, URL, 게시일, 확인일, 시행기간, 지역, 검토 상태, 수집 상태와 SHA-256을 가진다. 내용이 한 글자라도 바뀌었는데 해시를 갱신하지 않으면 서버 시작 시 검증에 실패한다.

| 정책 | 출처 상태 | 현재 처리 |
|---|---|---|
| 2026년 서울시 청년월세지원 | 공식 공고·사업개요 접근 확인 | `retrieved`, 사람 검토 대기 |
| 청년전용 보증부월세대출 | 공식 URL은 확인했으나 자동 접근 타임아웃 | `manual_snapshot`, 사람 검토 대기 |
| 주거안정월세대출 | 공식 URL은 확인했으나 자동 접근 타임아웃 | `manual_snapshot`, 사람 검토 대기 |

세 출처 모두 `review_status=pending`이다. 따라서 검색 결과와 인용은 제공하지만 상태는 `OFFICIAL_CHECK_NEEDED`이며, 정책 규칙은 계속 비활성화된다. 작성자와 다른 팀원이 원문을 확인하기 전에는 `approved`로 바꾸지 않는다.

## 3. 의미 단위 청킹

문서를 고정 글자 수로 무조건 자르지 않고 출처에서 정의한 제목과 구간을 우선한다.

- 지원대상
- 대상주택
- 지원내용·한도
- 신청기간·중복수혜
- 심사·이용절차

각 청크에는 `locator`와 관련 `condition_codes`를 함께 저장한다. 표의 행, 예외조건, 조건의 설명이 서로 다른 청크로 흩어지지 않게 하나의 정책 의미 단위로 관리한다. 긴 구간만 800자 한도에서 문단 경계를 기준으로 나눈다.

## 4. 하이브리드 검색

검색 점수는 다음 두 값을 결합한다.

- 키워드 점수 65%: 질문 토큰과 출처 토큰의 일치 비율
- 경량 벡터 점수 35%: 토큰을 고정 차원 해시 벡터로 바꾼 뒤 계산한 코사인 유사도

한국어 붙여쓰기 차이를 줄이기 위해 한글 단어의 2글자 토큰도 함께 사용한다. 해시 벡터는 외부 모델이 아니며 의미를 이해하는 임베딩보다 성능이 제한적이다. 대신 설치 용량과 비용이 없고 같은 입력에 항상 같은 결과를 낸다.

벡터 점수만으로는 후보를 만들지 않는다. 최소 하나의 정책 용어가 일치한 청크만 벡터로 재정렬하므로 무관한 질문이 해시 충돌 때문에 정책 답변으로 바뀌는 것을 막는다.

## 5. 메타데이터 필터와 안전 처리

검색 전에 다음 조건을 적용한다.

1. 요청한 정책 코드와 일치하는지 확인한다.
2. 요청 지역과 출처 지역이 맞는지 확인한다. `전국` 정책은 모든 지역에 적용한다.
3. 기준일이 정책 시행기간 안에 있는지 확인한다.
4. 허용된 공식 도메인인지 확인한다.
5. 스냅샷 해시가 실제 저장 내용과 일치하는지 확인한다.

적용 가능한 근거가 없으면 다음 결과를 반환한다.

```json
{
  "status": "OFFICIAL_CHECK_NEEDED",
  "reason_codes": ["NO_APPLICABLE_OFFICIAL_EVIDENCE"],
  "claims": [],
  "hits": []
}
```

즉 검색 실패 시 Qwen3-4B가 기억에 의존해 정책 내용을 채우지 않는다.

## 6. 문장별 인용

검색 결과는 원문 청크 전체와 별도로 질문에 가장 가까운 문장을 `quote`로 반환한다. 인용에는 다음 정보가 포함된다.

- 인용 ID와 청크 ID
- 정책 코드와 원문 문서 ID
- 제목, 기관과 공식 URL
- 문서 내 위치 `locator`
- 인용 문장 `quote`
- 확인일과 내용 SHA-256
- 수집 상태와 사람 검토 상태

모든 `GroundedClaim`은 하나 이상의 `citation_ids`를 가져야 한다. 근거 없는 문장은 모델 스키마상 생성할 수 없다.

## 7. 규칙 엔진 연결

`eligibility-with-evidence`는 Phase 4 규칙 결과를 바꾸지 않고 감싼다. 각 조건의 `condition_code`를 출처 청크의 `condition_codes`와 연결한다.

- 규칙 판정 결과
- 규칙 설명
- 통과·실패·누락 상태
- 해당 조건의 공식 출처 인용
- 근거 검토 상태

정책 규칙 또는 출처가 승인되지 않았거나 조건 근거가 누락되면 전체 근거 상태는 `OFFICIAL_CHECK_NEEDED`다. 기존 정책 규칙의 `document_version`, `rule_version`과 인용문의 `content_sha256`이 함께 응답되므로 어떤 규칙이 어떤 원문 스냅샷에 연결됐는지 재현할 수 있다.

## 8. API

```http
POST /policy-evidence/search
POST /policies/{policy_code}/eligibility-with-evidence
```

검색 요청 예시는 다음과 같다.

```json
{
  "question": "서울 청년 월세 지원금은 얼마인가요?",
  "as_of_date": "2026-05-10",
  "region": "서울특별시",
  "policy_codes": ["seoul_youth_monthly_rent_2026"],
  "top_k": 3
}
```

Phase 7 검색은 개인정보를 요구하지 않으므로 세션 토큰 없이 사용할 수 있다. 자격 판정 입력을 저장하는 기능은 추가하지 않았다.

## 9. 검색 평가

`evaluate_retrieval()`은 다음 지표를 별도로 계산한다.

- `hit_rate_at_k`: 정답 문서가 상위 K개 안에 검색됐는지
- `citation_alignment`: 검색 청크가 정답 조건 코드를 포함하는지

현재 합성 질문 3개 기준 Hit@3과 인용 정합성은 모두 100%다. 정책별 질문이 한 개뿐인 매우 작은 개발 평가셋이므로 실제 서비스 성능을 의미하지 않는다. 팀원 검토 후 경계조건, 유사 정책 혼동, 기간 만료, 지역 불일치 질문을 추가해야 한다.

## 10. 원문 변경·비활성화 절차

1. 공식 URL을 다시 확인한다.
2. 게시일, 확인일과 시행기간을 갱신한다.
3. 변경된 원문 구간과 SHA-256을 등록한다.
4. `changed_source_document_ids()`로 재청킹 대상 문서를 찾는다.
5. 관련 규칙 버전을 올리고 정책을 `draft`로 되돌린다.
6. 검색 평가와 규칙 경계 테스트를 다시 실행한다.
7. 작성자와 다른 팀원이 원문, 규칙, 인용 위치를 검토한다.
8. 검토가 끝난 버전만 `approved`로 전환한다.
9. 만료되거나 대체된 스냅샷은 `retired`로 남겨 과거 결과 재현에 사용한다.

자동 변경 감지는 향후 스케줄러가 공식 URL을 주기적으로 수집해 저장 해시와 비교하도록 연결한다. 현재 MVP에는 백그라운드 스케줄러를 추가하지 않았다.

## 13. 최종 누락 검토

핵심 기능과 안전 기능은 구현됐다. 다음 항목은 구현 누락이 아니라 운영 전 필수 검증으로 남는다.

1. 주택도시기금 두 정책의 공식 원문을 다시 내려받아 수동 스냅샷과 대조한다.
2. 세 정책의 조건, 예외, 시행기간과 인용 구간을 작성자 외 팀원 한 명이 검토한다.
3. 실제 사용자 질문을 정책별 최소 10개 이상 추가해 검색 평가를 다시 수행한다.
4. 정책 갱신 자동화는 서비스 배포 방식이 정해진 뒤 스케줄러에 연결한다.

이 검증 전까지 시스템은 의도적으로 `OFFICIAL_CHECK_NEEDED`를 반환하며 정책을 확정 추천하지 않는다.
