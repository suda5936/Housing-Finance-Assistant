# Phase 9 Web UI·의사결정 카드

## 1. 완료 범위

Phase 9에서는 Phase 2~8의 데이터·계산·정책·순위·오케스트레이션 API를 하나의 모바일 우선 사용자 흐름으로 연결했다.

```text
개인정보 동의
→ 프로필·예산 입력
→ 후보 2~3개 입력·수정·삭제
→ 선택 문서 업로드·추출값 대조
→ 계산 가정 확인
→ 비용·정책·순위·검증 실행
→ 의사결정 카드
→ 가중치 변경 재계산·데이터 다운로드·삭제
```

구현 상태는 **기능·자동검증 완료, 실제 모바일 브라우저 수동 확인 대기**다. Codex 브라우저 런타임이 이 PC의 `AppData` 접근 권한에서 시작되지 않아 화면 크기별 시각 검수만 사용자 확인 항목으로 남겼다.

## 3. 화면별 동작

### 3.1 동의와 세션

- 개인정보·민감정보 안내를 각각 확인해야 익명 세션을 만든다.
- 동의가 없으면 어떤 사용자 데이터도 전송하지 않는다.
- 세션 만료 시각을 표시하고 결과에서 JSON 다운로드와 즉시 삭제를 제공한다.
- API 오류는 원인만 알리는 대신 사용자가 취할 복구 행동을 함께 표시한다.

### 3.2 프로필과 후보

- 나이, 실수령액, 가용 보증금, 자산, 중위소득 비율, 금리, 초기비용과 정책 확인 정보를 입력한다.
- 데모 버튼은 실사용 정보가 아닌 가상 값을 한 번에 채운다.
- 후보는 2~3개로 제한하며 보증금, 월세, 관리비, 면적, 계약기간, 통근, 위험 신호와 근거를 입력한다.
- 저장된 후보는 같은 ID로 수정하고, 세 번째 후보는 서버와 화면에서 삭제할 수 있다.
- 후보 수정 후 기존 결과를 폐기하여 이전 계산이 잘못 재사용되지 않게 한다.

### 3.3 문서와 사용자 확정값

- PDF·PNG·JPEG, 최대 10MB 안내를 표시한다.
- 문서가 없어도 수동 입력 경로를 그대로 진행한다.
- 추출 상태, 필드, 정규화 값, 신뢰도와 인젝션 차단 상태를 표시한다.
- OCR 추출값은 자동 계산에 반영하지 않는다. 사용자는 `후보 입력과 대조·수정` 버튼으로 돌아가 직접 확정한다.
- 주민번호·연락처 마스킹, 문서 지시문 미실행과 삭제 원칙을 업로드 전에 보여준다.

## 4. 계산·에이전트 연결

화면은 다음 순서로 실제 API를 호출한다.

1. 후보별 낙관·기준·보수 비용 계산
2. 선택한 가중치로 후보 순위 계산
3. 세션에 에이전트 실행 생성
4. 결정론적 상태 머신 자동 진행
5. 최종 검증 게이트 결과 표시

Qwen3-4B는 금액, 정책 상태나 순위를 만들거나 수정하지 않는다. 현재 핵심 화면은 검증된 규칙 함수 결과만 표시하므로 Ollama가 중단돼도 수동 입력과 결정론적 비교가 가능하다.

## 5. 의사결정 카드

최종 화면은 다음 항목을 한 카드 안에서 제공한다.

- 현재 가중치에서 1위인 후보와 추천 근거
- 월평균 실질 주거비와 계약기간 총비용
- 소득 대비 주거비 부담률
- 가장 큰 순위 기여 기준
- 전체 후보의 비용·통근·면적·보증금·위험 신호 비교
- 기준별 기여도와 후보 간 트레이드오프
- 균형·비용 우선·통근 우선 프리셋 재계산
- 결과가 바뀌는 조건과 정책 판정 상태
- 계약 전 확인 체크리스트와 항목별 공식 출처
- 계산 버전, 입력 해시, 기준일과 계산 가정
- 서비스 한계, 공식기관 재확인 안내

정책 원문 교차검토 전에는 성공처럼 보이는 초록색 결과 대신 텍스트와 아이콘을 포함한 `공식 확인 필요` 상태로 종료한다.

## 6. 접근성·반응형 기준

- `lang="ko"`, 스킵 링크, 단계별 제목 자동 포커스를 적용했다.
- 입력마다 연결된 레이블을 제공하고 진행·오류 상태에는 `aria-live` 또는 `role="alert"`를 사용한다.
- 모든 조작은 기본 버튼·입력·상세 요소로 키보드 접근 가능하다.
- 포커스 링, 최소 45px 조작 영역, 고대비 본문 색상과 모션 감소 설정을 적용했다.
- 정책·성공·주의 상태는 색상뿐 아니라 아이콘과 문구로 구분한다.
- 900px과 620px 분기에서 열 수, 단계 표시, 버튼 배치와 결과 표 스크롤을 조정한다.

## 7. 오류와 복구

| 상황 | 화면 처리 | 복구 행동 |
|---|---|---|
| 동의 누락 | 세션 생성 차단 | 두 안내 확인 |
| 프로필·후보 누락 또는 범위 오류 | 첫 오류 필드 설명 | 표시된 값 수정 |
| 후보가 2개 미만 | 삭제 차단 | 후보 값 수정 또는 추가 |
| 문서 형식·크기·OCR 오류 | 분석 중단 없이 오류 안내 | 문서 없이 진행 또는 파일 교체 |
| API 연결 실패 | 구조화 오류 배너 | 로컬 API 확인 후 재시도 |
| 정책 검토 미완료 | 추천 확정 표현 차단 | 공식 출처와 담당기관 확인 |
| 세션 만료 | 저장·분석 차단 | 첫 단계에서 새 세션 생성 |

## 8. 개인정보 제어

- 사용자는 결과 JSON을 내려받을 수 있다.
- `세션과 데이터 삭제`는 서버의 세션·프로필·후보·문서와 로컬 화면 상태를 함께 삭제한다.
- 문서 원본은 익명 세션에만 연결하고 기본 보관기간은 24시간이다.
- 화면에서 주민번호, 계좌, 연락처 입력을 요구하지 않는다.

## 10. 사용자 수동 점검

```powershell
npm run dev
```

1. `http://localhost:3000`을 연다.
2. 개발자 도구에서 화면 폭을 390px로 설정한다.
3. 동의 → `데모 값 채우기` → 후보 저장 → 문서 없이 진행 → 분석을 완료한다.
4. 결과에서 가중치 프리셋을 바꾸고 순위 또는 조건 문구가 갱신되는지 확인한다.
5. 후보 수정, JSON 다운로드, 세션 삭제를 확인한다.
6. 마우스 없이 `Tab`, `Shift+Tab`, `Enter`, `Space`로 동일 흐름을 확인한다.

이 점검에서 화면 잘림이나 조작 불가가 없으면 로드맵의 모바일 완료 조건을 체크하고 Phase 10으로 진행한다.

- 현재 메모리 저장소를 운영 영속 저장소로 바꿀 때 전송·저장 암호화와 접근제어를 함께 적용한다.
- 문서 만료 삭제를 운영 스케줄러에 연결하고 삭제 감사 이벤트를 남긴다.
- 업로드 파일 시그니처와 콘텐츠 검사에 악성코드 검사 계층을 추가한다.
- 요청 상관관계 ID, 단계별 지연시간, 오류·검증 실패율을 구조화 로그로 수집한다.
- Phase 7 정책 3개의 팀원 독립 검토가 끝나기 전에는 `official_check` 게이트를 해제하지 않는다.
