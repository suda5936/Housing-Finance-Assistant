# 집결정 AI Phase 3 — 주거비 계산 엔진

## 1. 완료 결과

- 상태: **완료**
- 기준일: 2026-07-24
- 계산 버전: `cost-v1`
- 계산 방식: Python `Decimal` 기반 순수 함수
- LLM 사용: 없음
- API: `POST /costs/calculate`
- 시나리오: 낙관·기준·보수
- 통근 데이터: 무료 MVP에서는 사용자 수동 입력, 지도 출처 메타데이터 형식은 지원

Phase 3은 추천 순위를 결정하지 않는다. 후보 한 개의 실질 비용과 부담률을 동일한 계산식으로 산출하며, 여러 후보 비교는 Phase 5에서 이 결과를 입력으로 사용한다.

## 2. 확정한 계산 원칙

1. 기준소득은 월 세후소득이다.
2. 보증금 원금은 반환 가능한 금액이므로 비용에서 제외한다.
3. 차입한 보증금에는 연이율을 월 이자비용으로 환산해 반영한다.
4. 자기자금 기회비용은 사용자가 입력한 연이율이 0보다 클 때만 반영한다.
5. 초기비용은 계약기간 개월 수로 나눠 월평균 비용에 반영한다.
6. 관리비가 없다는 뜻의 0원과 관리비를 모른다는 `null`을 구분한다.
7. 통근비는 월 비용 직접 입력 또는 왕복 비용 × 월 출근일수로 계산한다.
8. 확정 지원금은 모든 시나리오에서 반영한다.
9. 선정 전 예상 지원금은 낙관 시나리오에서만 반영한다.
10. 지원금은 반복 주거비보다 많이 차감할 수 없다.
11. 모든 계산은 반올림 전 Decimal 중간값을 사용하고, 표시 금액만 원 단위 반올림한다.
12. 같은 입력은 같은 입력 해시와 같은 계산 결과를 반환한다.

## 3. 계산 공식

### 3.1 보증금 월 조달비용

```text
차입 보증금 = 보증금 - 자기자금 보증금

월 차입비용
= 차입 보증금 × 연 차입금리 ÷ 100 ÷ 12

월 자기자금 기회비용
= 자기자금 보증금 × 연 기회비용률 ÷ 100 ÷ 12
```

### 3.2 월 통근비

월 비용을 직접 입력한 경우:

```text
월 통근비 = 입력한 월 통근비
```

왕복 비용을 입력한 경우:

```text
월 통근비 = 왕복 1회 비용 × 월 출근일수
```

### 3.3 계약기간 반복비용

```text
월 반복비용
= 월세
+ 관리비
+ 월 차입비용
+ 월 자기자금 기회비용
+ 월 통근비
+ 선택 생활비
```

```text
계약기간 반복비용 = 월 반복비용 × 계약개월
```

### 3.4 적용 지원금

```text
정책별 적용개월
= min(지원기간, 계약개월 - 지원시작월 + 1)
```

지원 시작월이 계약기간 뒤라면 적용개월은 0이다.

```text
적용 지원금
= 월 지원금 × 적용개월
```

### 3.5 실질 비용과 부담률

```text
계약기간 총비용
= 계약기간 반복비용
+ 일회성 초기비용
- 적용 지원금
```

```text
월평균 실질 주거비
= 계약기간 총비용 ÷ 계약개월
```

```text
주거비 부담률
= 월평균 실질 주거비 ÷ 월 세후소득 × 100
```

## 4. 시나리오 규칙

| 입력 | 낙관 | 기준 | 보수 |
| --- | --- | --- | --- |
| 관리비 범위 | 최소 | 기준 | 최대 |
| 차입 금리 범위 | 최소 | 기준 | 최대 |
| 자기자금 기회비용률 | 최소 | 기준 | 최대 |
| 확정 지원금 | 포함 | 포함 | 포함 |
| 예상 지원금 | 포함 | 제외 | 제외 |

관리비와 금리가 모두 단일값이면 세 시나리오의 해당 항목은 동일하다. 예상 지원을 반영한 낙관 결과는 수혜를 보장하는 결과가 아니며 화면에서 반드시 `지원 선정 전 가정`으로 표시한다.

## 5. 입력 스키마

주요 입력:

- 후보 이름
- 후보 지역(구 단위)
- 월 세후소득
- 보증금과 그중 자기자금
- 월세
- 최소·기준·최대 관리비
- 최소·기준·최대 연 차입금리
- 최소·기준·최대 자기자금 기회비용률
- 계약기간
- 중개비·이사비 등 초기비용 합계
- 통근비·통근시간·교통수단·기준시각
- 선택 생활비
- 확정 또는 예상 지원금과 적용기간

모든 금액은 다음 형식으로 전달한다.

```json
{
  "amount": "550000.00",
  "currency": "KRW"
}
```

입력 제한:

- 월 세후소득은 0보다 커야 한다.
- 자기자금 보증금은 전체 보증금을 넘을 수 없다.
- 계약기간은 1~120개월이다.
- 금리는 0~100% 범위다.
- 관리비·금리 범위는 `최소 ≤ 기준 ≤ 최대`여야 한다.
- 통근비는 월 비용과 왕복 비용 중 하나만 입력한다.
- 왕복 비용 방식에는 월 출근일수가 필요하다.
- 음수 금액은 허용하지 않는다.

## 6. 출력 스키마

```text
CostCalculationResponse
├─ status
├─ calculation_version
├─ input_sha256
├─ missing_fields
└─ scenarios[]
   ├─ scenario
   ├─ breakdown
   │  ├─ monthly_rent
   │  ├─ monthly_maintenance
   │  ├─ monthly_borrowing_cost
   │  ├─ monthly_opportunity_cost
   │  ├─ monthly_commute_cost
   │  ├─ monthly_living_cost
   │  ├─ monthly_initial_cost_amortization
   │  ├─ monthly_support_average
   │  ├─ monthly_effective_cost
   │  ├─ contract_total_cost
   │  └─ housing_cost_burden_percent
   ├─ assumptions
   └─ reason_codes
```

상태:

- `calculated`: 필수값이 충분하여 세 시나리오 계산 완료
- `missing_information`: 계산에 필요한 관리비 또는 통근정보 부족

관리비와 통근정보가 없으면 0원으로 추정하지 않고 `missing_fields`를 반환한다.

## 7. 결과 사유 코드

| 코드 | 의미 |
| --- | --- |
| `REFUNDABLE_DEPOSIT_EXCLUDED` | 반환 가능한 보증금 원금을 비용에서 제외 |
| `MAINTENANCE_RANGE_SCENARIO_APPLIED` | 관리비 범위별 시나리오 적용 |
| `OWN_FUNDS_OPPORTUNITY_COST_INCLUDED` | 자기자금 기회비용 반영 |
| `MANUAL_COMMUTE_INPUT_USED` | 사용자가 입력한 통근정보 사용 |
| `EXPECTED_SUPPORT_INCLUDED` | 예상 지원금을 낙관 시나리오에 반영 |
| `EXPECTED_SUPPORT_EXCLUDED` | 예상 지원금을 기준·보수 시나리오에서 제외 |
| `SUPPORT_CAPPED_AT_RECURRING_COST` | 지원금 차감액이 반복비용을 넘지 않도록 제한 |

LLM은 이 사유 코드를 설명할 수 있지만 사유 코드나 계산값을 새로 만들 수 없다.

## 8. API 계약

```http
POST /costs/calculate
Content-Type: application/json
```

이 API는 순수 계산 API이므로 세션 토큰이 필요 없고 데이터를 저장하지 않는다. Phase 8 에이전트는 세션에서 사용자 확정값을 읽어 이 API와 같은 계산 함수를 호출한다.

## 9. 통근 데이터와 무료 MVP 범위

Phase 0에서 지도 유료 API를 사용하지 않기로 했으므로 MVP 기본 경로는 수동 입력이다.

수동 입력 항목:

- 교통수단
- 편도 통근시간
- 월 통근비 또는 왕복 비용
- 월 출근일수
- 입력·조회 기준시각

`source=map` 형식도 지원하여 추후 승인된 지도 API 결과를 동일 스키마로 받을 수 있다. 지도 API가 없거나 실패하면 `source=manual`로 바꾸고 같은 계산 엔진을 사용한다. 정확한 주소는 계산 요청에 포함하지 않고 구 단위 후보 정보만 별도로 관리한다.

## 11. Phase 4 전달사항

1. 정책 규칙 엔진은 지원금 상태를 `confirmed`와 `expected`로 구분한다.
2. 지원금 결과에는 정책 문서·규칙 버전을 함께 전달한다.
3. 실제 선정 전 지원금은 기본 비용에서 차감하지 않는다.
4. 정책 적용 시작월과 지원기간을 명시한다.
5. Phase 4 판정 결과를 Phase 3 `SupportItem`으로 변환하는 매퍼를 별도로 둔다.
