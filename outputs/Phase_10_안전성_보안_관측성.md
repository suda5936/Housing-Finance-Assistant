# Phase 10 안전성·보안·관측성

## 1. 완료 범위

Phase 10에서는 Phase 2~9의 보호 기능을 다시 감사하고, 로컬 MVP에서 검증 가능한 요청 경계·외부 전송·로그·만료·추적 통제를 보강했다.

```text
허용 Host·요청 크기 검사
→ 상관관계 ID 부여
→ 입력 스키마·세션 토큰 검증
→ 업로드 시그니처·활성 요소 검사
→ 결정론적 도구 실행·검증 게이트
→ 민감정보 제거 구조화 로그
→ 집계 지표와 감사 기록
→ 즉시 삭제 또는 만료 정리
```

구현 상태는 **MVP 애플리케이션 통제 완료**다. TLS, 저장 데이터 암호화 키, 악성코드 엔진, 관리자 RBAC, 비밀 저장소와 인프라 모니터링은 배포 플랫폼을 정한 뒤 연결해야 하는 운영 전 필수 항목으로 분리했다.

- 계산 결과, 후보 순위와 에이전트 감사 기록
- 로컬 Qwen3-4B 및 OCR 실행 경계

2. FastAPI ↔ 로컬 파일 저장소

5. 애플리케이션 로그·지표 ↔ 운영자

## 3. 요청 경계 보안

`RequestSafetyMiddleware`는 모든 정상 API 요청에 다음을 적용한다.

- 사용자 제공 `X-Request-ID`는 8~64자의 제한된 문자만 허용하고, 나머지는 UUID로 교체한다.
- `Content-Length`가 `MAX_REQUEST_BYTES`를 넘으면 본문 처리 전에 `413 REQUEST_TOO_LARGE`로 거부한다.
- 응답에 `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, `Referrer-Policy: no-referrer`를 추가한다.
- 카메라·마이크·위치 권한을 `Permissions-Policy`로 차단한다.
- 세션 경로에는 `Cache-Control: no-store`를 추가한다.
- HTTPS 배포에서는 `ENABLE_HSTS=true`로 HSTS를 켤 수 있다. 로컬 HTTP에서는 기본적으로 끈다.
- `TRUSTED_HOSTS` 외의 Host 요청을 400으로 거부한다.

앱 제한만으로 네트워크 계층 공격을 완전히 막을 수 없으므로 배포 시 리버스 프록시에도 요청 본문 제한과 요청률 제한을 적용해야 한다.

## 4. 파일·문서 안전

기존 Phase 6 기능을 재검토한 결과 다음 통제가 실제 코드와 테스트에 존재한다.

- PDF·PNG·JPEG만 허용
- 매직바이트, MIME과 확장자의 3중 일치 확인
- 파일명 경로 포함 거부와 저장 경로 루트 이탈 방지
- 최대 10MB, PDF 10페이지, 이미지 2,500만 픽셀
- 활성 PDF 요소 거부
- 파서·OCR 시간 제한과 실패 시 수동 입력 전환
- 주민번호·계좌·전화 등 추출 텍스트 마스킹
- 문서 인젝션 탐지 후 명령이 아닌 데이터로 처리

ClamAV 같은 전문 악성코드 스캐너는 현재 로컬 의존성을 늘리지 않기 위해 포함하지 않았다. 공개 배포의 파일 업로드 활성화 전에는 필수다.

## 5. 개인정보와 외부 공급자

- 세션 토큰은 원문이 아닌 SHA-256 해시로 저장하고 `hmac.compare_digest`로 확인한다.
- 로그는 주민번호, 계좌, 전화번호, 이메일을 문자열에서 마스킹한다.
- `access_token`, `session_token`, `x-session-token`, `api_key`, `password`, `cookie`, `prompt`, `user_prompt`, `document_text` 키의 값은 전부 제거한다.
- Qwen3-4B는 `localhost` 또는 loopback IP의 Ollama만 허용한다. 원격 주소가 설정되면 호출하지 않고 수동 경로로 축소한다.
- 세션 삭제는 문서 원본, 메타데이터, 프로필, 후보, 분석 스냅샷과 등록된 에이전트 캐시를 함께 제거한다.
- 서버 시작 시 만료 정리 루프가 실행되어 기본 5분마다 만료 세션과 파일을 제거한다.

## 7. 관측성

### 7.1 요청 추적

모든 처리 요청은 동일한 `X-Request-ID`를 다음 위치에서 공유한다.

- API 응답 헤더
- 구조화 오류 응답의 `error.request_id`
- `request_completed` 로그

로그에는 메서드, 템플릿화된 라우트, 상태 코드와 처리시간만 기록하며 요청 본문·세션 ID·후보 ID는 기록하지 않는다.

### 7.2 집계 지표

`GET /system/metrics`는 프로세스 메모리에서 다음 비식별 지표를 제공한다.

- 전체 요청·오류 수
- 라우트 템플릿별 요청·오류 수
- 평균·최대 처리시간

UUID 실값 대신 `/sessions/{session_id}/...` 템플릿을 사용해 개인정보 노출과 고카디널리티를 막는다. 현재 지표는 재시작 시 초기화되는 개발용이다. 운영에서는 인증된 내부 경로로 제한하고 Prometheus/OpenTelemetry 등에 전송한다.

Phase 8의 `AgentRun`에는 도구별 입력·출력 해시, 상태, 시간, 캐시·수동전환, 호출 횟수와 LLM 토큰·추정비용이 별도로 기록된다.
